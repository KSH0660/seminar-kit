LLM의 토큰 흐름

Prefill vs Generation — 모든 단계의 공통 원리

구분 의미 학습 Loss
Prefill 시스템이 컨텍스트에 주입 — 모델이 생성하지 않음 OFF
Generation 모델이 학습을 통해 스스로 출력 ON

Stage 1 — Chat

<|system|> 시스템 프롬프트 <|user|> 사용자 질문 <|assistant|> 자연어 답변 <|endoftext|>

▶ 프레임워크가 <|assistant|>까지 주입(Prefill) → 모델이 이어서 생성(Generation)

LLM은 본질적으로 Generation 기계 — 프레임워크가 조건(Prefill)을 세팅하고, 모델이 빈칸을 이어 쓴다

그렇다면 추론(Reasoning)과 도구 호출(Tool Calling)에서는?

Generation 영역의 확장

Chat → Reasoning → Tool-Integrated Reasoning

Chat

<|system|> ... <|user|> 질문 <|assistant|> 자연어 답변 <|endoftext|>

Reasoning

<|system|> ... <|user|> 질문 <|assistant|> <|think|> 추론 과정 (CoT) 자연어 답변 <|endoftext|>

Tool-Integrated

<|system|> ... <|user|> 질문 <|assistant|> <|think|> 추론 과정 <|tool_call|> func(args) <|observation|> 도구 실행 결과 최종 답변 <|endoftext|>
<|think|>, tool_call JSON, 자연어 답변, <|endoftext|> — 모두 Generation (Loss ON)

Special token이 늘어도 구조는 동일 — Prefill → <|assistant|> → Generation
