# 디자인 가이드

## 철학

**지루한 슬라이드를 만들지 마라.** 흰 배경에 bullet만 나열하면 아무도 기억하지 못한다.

---

## 시작 전 체크

- **주제에 맞는 컬러 팔레트 선택** — 기본 파란색에 안주하지 않기
- **60-30-10 비율** — 주색 60%, 보조색 30%, 강조색 10%
- **다크/라이트 대비** — 타이틀·결론은 다크, 본문은 라이트 ("샌드위치 구조")
- **하나의 시각적 모티프** — 둥근 프레임, 아이콘 원형 배경, 두꺼운 좌측 border 등 하나를 골라 전체에 반복

## 슬라이드별 레이아웃 아이디어

**모든 슬라이드에 시각적 요소가 있어야 한다** — 텍스트만 있는 슬라이드는 잊힌다.

- **2열 레이아웃**: 텍스트 좌 + 비주얼 우
- **아이콘 + 텍스트 행**: 원형 배경 아이콘, 볼드 헤더, 설명
- **큰 수치 강조**: 60-72pt 숫자 + 작은 라벨
- **비교 컬럼**: before/after, pros/cons
- **타임라인/프로세스**: 번호 + 화살표 흐름
- **풀 이미지**: 한쪽 전체 이미지 + 반대쪽 텍스트

---

## 타이포그래피

| 요소 | 권장 크기 |
|------|-----------|
| 슬라이드 제목 | 1.6em (약 45px) |
| 섹션 헤더 | 1.1em (약 31px) |
| 본문 | 1em (28px, 기본) |
| 보조 텍스트 | 0.85em (약 24px) |
| 각주 | 0.65em (약 18px) |

## 피해야 할 것

- **같은 레이아웃 반복** — 컬럼, 카드, 콜아웃을 다양하게
- **본문 가운데 정렬** — 제목만 가운데, 본문은 좌정렬
- **크기 대비 부족** — 제목과 본문 크기 차이가 확실해야
- **기본 파란색 고수** — 주제에 맞는 색상 선택
- **텍스트만 있는 슬라이드** — 이미지, 아이콘, 차트 등 시각 요소 필수
- **제목 아래 강조선** — AI 생성 느낌의 대표적 안티패턴. 여백이나 배경색으로 대체

## reveal.js 테마 변경 방법

1. `latest.html`에서 테마 CSS 변경:
   ```html
   <!-- white.css → black.css, moon.css, night.css 등 -->
   <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/reveal.js@5.1.0/dist/theme/black.css">
   ```
2. `styles.css`의 `:root` 변수를 새 테마에 맞게 조정
3. 폰트 변경: `@import` URL + `.reveal` `font-family` 수정
